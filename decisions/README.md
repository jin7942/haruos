# 의사결정 기록 (RFC → ADR)

프로젝트의 주요 의사결정을 한 곳에서 관리한다.

```
   아이디어/문제
        │
   ┌────▼─────┐   논의가 필요한가? (대안 다툼·합의 필요·되돌리기 비쌈)
   │   RFC    │
   │ 제안→논의 │
   │ 중→합의   │
   └────┬─────┘
        │  합의됨 → "확정으로 승격"
   ┌────▼─────┐
   │   ADR    │   확정. 본문 불변. 번복 시 새 ADR로 대체.
   │   확정    │
   └──────────┘
```

- **RFC** (`RFC/`) = 결정에 이르는 토론. 여러 선택지를 펼치고 합의를 모은다. 살아 움직이는 문서.
- **ADR** (`ADR/`) = 내려진 결정. 합의 결과를 박제한다. 확정 후 본문 수정 금지.
- 작고 자명한 결정(대안이 하나, 합의 불필요, 되돌리기 쌈)은 RFC 없이 **바로 ADR**.
  대안이 다투거나 합의가 필요한 결정만 RFC를 거친다.

## 폴더 구조

```
decisions/
├── README.md            # 이 파일 (의사결정 인덱스 + 프로세스)
├── ADR/
│   ├── README.md        # ADR 목록 표
│   └── ADR-NNN-제목.md
└── RFC/
    ├── README.md        # RFC 목록 표 + 상태 정의
    ├── TEMPLATE.md      # RFC 복사 시드
    └── RFC-NNN-제목.md
```

## RFC → ADR 승격 규칙

RFC가 `합의` 상태에 도달하면 ADR로 승격한다.

1. **ADR 신규 작성** — RFC의 *추천 옵션*을 `결정`으로, *검토 선택지/근거*를 ADR
   템플릿에 압축 이식. 상세 토론은 RFC 링크로 위임.
2. **ADR `관련 RFC` 칸에 원본 RFC 링크** 명시.
3. **RFC 상태를 `확정으로 승격`으로 변경**, RFC 상단 `관련 ADR`에 새 ADR 링크.
4. **RFC 원문은 보존** (삭제 금지 — 논의 맥락은 ADR보다 풍부한 1차 사료).
5. RFC의 *미해결 질문* 중 무거운 항목은 *후속 RFC로 분리*해 별도 승격.

승격하지 않는 경로:
- RFC `폐기` → 채택 안 함 (단, "왜 안 했나"가 가치 있으면 ADR로 박제 가능).
- RFC `보류` → 재검토 대기. 트리거 충족 시 `논의 중`으로 복귀.

## 작성 트리거

- **RFC를 쓴다**: 아키텍처/BM 방향 등 대안이 다투거나, 합의가 필요하거나,
  되돌리기 비싼데 아직 확신이 없거나, 미해결 질문이 여럿인 결정.
- **바로 ADR**: 대안이 자명하고 합의 불필요, 되돌리기 비용이 작은 결정.
- **둘 다 안 씀**: 일상 버그 수정·리팩터링·1~3줄 변경·단순 문서 갱신 → `.claude-log/`.

자세한 ADR/RFC 작성 기준·템플릿은 `CLAUDE.md`의 "의사결정 기록 (RFC → ADR)" 절 참조.
