# ADR-004: 무료 사용자 격리 전략 (이중 트랙 Pool + Silo)

- **상태**: 확정
- **일자**: 2026-06-12
- **결정자**: 김진범
- **관련 RFC**: [RFC-001 무료 사용자 격리 전략 (Pool vs Silo)](../RFC/RFC-001-무료-사용자-격리-전략.md)

## 배경

HaruOS는 현재 사용자 본인 AWS 계정에 테넌트 인프라를 프로비저닝하는 단일
모델(Silo)만 지원한다. 두 가지 문제가 있다.

1. **진입 장벽**: 가입에 AWS 계정·결제카드·Role 설정이 필요해 무료 체험자·SMB·
   개인 수요를 전혀 못 잡는다. "$19/월" 가격표가 실제 $120+ TCO를 숨긴다.
2. **수익화 허점**: 구독 취소 시 인프라 정리 호출이 없고(`billing.service.ts:147`),
   tenant-api에 라이선스 체크가 0건이라, trial 중 인프라만 챙기고 이탈 가능.

코드베이스가 골격 단계라 DB 스키마·인증 흐름을 손대기 가장 좋은 시점이다.

상세 분석은 RFC-001 참조.

## 검토 선택지

1. **옵션 1 — 현행 유지 (Silo only, BYOC)**
   → 구현 변경 없음·데이터 주권 강점. 그러나 무료 경로 부재로 깔때기 위쪽이
   비좁고 수익화 허점이 그대로. SMB 시장 포기.
2. **옵션 2 — 이중 트랙 (Pool + Silo)** *(채택)*
   → 체험·무료는 본사 AWS 공유 인프라에 논리 격리(`tenant_id` + RLS), 정식 계약
   기업만 고객사 AWS에 Silo. 진입 장벽 0, 검증된 패턴, Silo 자산 보존.
   대신 본사 AI 비용 부담·멀티테넌트 격리 보안 책임·DB 스키마 변경 필요.
3. **옵션 3 — Pool only (모두 본사 AWS)**
   → 단일 모델로 단순하나 **데이터 주권 컨셉(ADR-001)과 정면 배치**, 규제 산업·
   엔터프라이즈 영업 불가, AI 비용 전체를 본사가 부담.

## 결정

**옵션 2 — 이중 트랙(Pool + Silo)을 채택한다.** 무료/체험 사용자는 본사 AWS에
논리적 격리(`tenant_id` + PostgreSQL RLS)로, 정식 계약 기업은 현행 Silo(고객사
AWS 물리적 격리)로 제공한다.

## 근거

1. GitLab·Sentry·Mattermost·Cal.com·n8n 등 다수 성공 사례로 검증된 패턴.
2. 기존 Silo 코드 자산을 폐기하지 않으면서 SMB 신규 시장 진입.
3. Pool은 본사 인프라라 사용량 한도 강제가 가능해 수익화 허점을 닫는다.
4. ADR-001(데이터 주권 = 핵심 가치)을 Silo 트랙에서 그대로 유지하며 확장.
5. 코드 골격 단계라 마이그레이션 부담이 최소.

## 영향

- **코드**: tenant-api 전 테이블에 `tenant_id` + RLS, pgvector 검색에도
  `tenant_id` 필터 강제(누락 시 타 회사 문서 노출 — 치명적), 사용량 한도
  미들웨어 신설, 백업/Export 완성, console-api 배포 모드 분기(Pool 등록 vs Silo
  프로비저닝), 가입 흐름 분리.
- **인프라**: 본사 AWS에 Pool 트랙용 tenant-api 1세트(ECS+RDS 공유) + 운영 도구.
- **문서·정책**: 무료/유료 기능 매트릭스, 이용약관(Pool 데이터 처리), 가격 페이지 분리.
- **영향 없음**: ADR-001은 Silo 트랙에서 유지. Stripe 결제 흐름은 Pool 유료 전환에 재사용.

### 미확인 사항 (후속 RFC로 분리)

RFC-001 §미해결 질문의 다음 항목은 별도 RFC로 결정한다:
무료 티어 한도, AI 모델 제한, DB 멀티테넌시 구현 방식, 유료 전환 트리거,
체험→엔터프라이즈 마이그레이션, 엔터프라이즈 가격, 무료 인증 강도, 본사 SLA,
법적 책임, BSL 라이센스 영향.
