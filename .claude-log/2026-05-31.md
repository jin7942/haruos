## 13:25 - [fix] 192.168.0.66 접속 불가 (502) 해결

### 요청 내용
192.168.0.66 으로 접속 안됨. nginx 리버스 프록시 구조 아닌가?

### 원인 (3개 별개 버그)
1. Vite dev 서버가 localhost(127.0.0.1)에만 바인딩
   - compose `dev -- --host`의 `--`가 인자를 끊어 `--host`가 vite에 전달 안 됨
   - nginx(별도 컨테이너)가 console-web-dev:5173 에 못 붙음 → 502
2. console-api 부팅 크래시: StripeAdapter 생성자에서 `new Stripe('')`
   - STRIPE_SECRET_KEY 미설정 시 Stripe SDK가 즉시 throw → DI 실패 → API 다운
3. console-api 부팅 크래시: TypeORM DataTypeNotSupportedError
   - `string | null` 유니온 + length 컬럼에 `type` 누락 → reflect-metadata가 Object 추론
   - provisioning-job(2), tenant-infra(11), domain(5) 엔티티 = 18개 컬럼

### 수행한 작업
- apps/console-web/vite.config.ts, tenant-web/vite.config.ts: `server.host: true` 추가
- infra/docker/docker-compose.dev.yml: web-dev command 의 `-- --host` 제거
- stripe.adapter.ts: 키 없으면 placeholder로 초기화 + warn (지연 실패)
- provisioning-job/tenant-infra/domain entity: nullable varchar 컬럼에 `type: 'varchar'` 명시

### 결과 (성공)
- 192.168.0.66:5173, :5174 → 200 (web), /api/ → 404 (api 정상 도달)
- console-api 정상 부팅 (BOOT_OK)
- console-api 테스트 153개 전부 통과 (회귀 없음)

### 교훈
- TypeORM 엔티티에서 `string | null` 유니온 컬럼은 `type` 명시 필수
- jest 단위테스트 통과 != 런타임 부팅 성공 (DB 연결/DI 검증 안 됨)
- 외부 키 미설정에도 앱은 부팅돼야 함 (어댑터 생성자에서 즉시 throw 금지)
