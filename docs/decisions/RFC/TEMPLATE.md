# RFC-NNN: 제목

| 버전 | 변경내용 | 작성자 | 수정일 |
| --- | --- | --- | --- |
| v1.0 | 초기 작성 | 이름 | YYYY-MM-DD |

- **상태**: 제안 / 논의 중 / 합의 / 확정으로 승격 / 폐기 / 보류
- **일자**: YYYY-MM-DD
- **작성자**: 이름
- **관련 ADR**: (확정 승격 시 ADR 링크)

## 요약

한두 단락으로 무엇을 제안하는지.

## 배경

왜 이 논의가 필요한가. 현재 어떤 문제·요구·기회가 있는가.

## 검토 선택지

각 선택지의 장단점.

1. **선택지 A**
   - 장점:
   - 단점:
2. **선택지 B**
   - 장점:
   - 단점:

## 추천

어떤 선택지를 추천하는지 + 근거. (강한 의견이 없으면 "추천 없음 — 팀 논의 필요"로 명시)

## 미해결 질문

팀 논의로 결정해야 할 것들. 답이 나오기 전까지 ADR로 승격할 수 없는 항목.

- [ ] 질문 1
- [ ] 질문 2

## 영향

이 제안이 채택되면 무엇이 바뀌어야 하는가. (코드 / 문서 / 인프라 / 운영 절차)

## 다음 단계
